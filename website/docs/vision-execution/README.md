# Vision execution docs

Progressive-disclosure map for agents working on the Ohana redesign. Start with `../../VISION_EXECUTION.md`; only open the deeper files when the task needs them.

## Files

- [`current-status.md`](current-status.md) — delivered state, active sprint, open follow-ups, deferred scope.
- [`sprint-plan.md`](sprint-plan.md) — sprint-level plan and acceptance criteria. Read only when planning or selecting the next slice.
- [`review-index.md`](review-index.md) — links to CEO/PM/QA/principal-engineer reports that may need follow-up.
- [`archive/progress-log-2026-07-12-13.md`](archive/progress-log-2026-07-12-13.md) — historical slice log. Do not read for routine implementation.
- [`archive/steering-reviews-2026-07-13.md`](archive/steering-reviews-2026-07-13.md) — historical steering snapshots. Do not read unless auditing decision history.

## Maintenance rule

Keep the top-level tracker short. Move completed slice logs, stale steering notes, and old acceptance criteria into archive files instead of appending forever.
